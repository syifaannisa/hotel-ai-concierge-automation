# Contributing

Thank you for your interest in improving this project.

This repository is maintained as a professional portfolio project and technical demonstration. Documentation improvements, bug reports, implementation suggestions, and secure integration ideas are welcome.

## Guidelines

1. Create a branch from `main`.
2. Keep each change focused and clearly documented.
3. Never include API keys, credentials, private URLs, spreadsheet identifiers, or personal data.
4. Use clear professional English for documentation, node names, and commit messages.
5. Test imported n8n workflows before proposing changes.
6. Update the relevant documentation when behaviour changes.

## Suggested commit style

```text
docs: improve setup instructions
fix: correct Telegram session routing
feat: add WhatsApp channel template
security: remove exposed configuration value
```

## Pull requests

Explain the business or technical problem, the proposed change, how it was tested, and any configuration impact.
