# Portfolio Screenshot Guide

Actual screenshots are not included because the source package did not contain image files. Capture the following views after importing the sanitised workflows into a safe demonstration n8n environment.

## Recommended image order

1. **Cover image**  
   A clean architecture diagram with the project title and the three supported channels.

2. **Website and Tally workflow overview**  
   Show the complete canvas at a readable zoom. Hide browser bookmarks, account details, execution URLs, and credential names.

3. **Telegram guest journey**  
   Show welcome menu, booking questions, booking summary, and confirmation using fictional guest details.

4. **Telegram workflow overview**  
   Emphasise the session-state logic and Google Sheets integration.

5. **Email guest journey**  
   Show a fictional incoming enquiry and the AI-assisted response. Hide all personal addresses.

6. **Email workflow overview**  
   Show trigger, parsing, classification, AI response, Gmail reply, and logging.

7. **Shared Google Sheets workspace**  
   Show only fictional sample records. Blur or remove document identifiers from the browser address bar.

## Visual standards

- Use English captions.
- Use consistent 16:9 or 4:3 framing.
- Crop empty browser space.
- Avoid exposing tokens, credentials, webhook URLs, account names, execution IDs, or personal information.
- Use the same fictional hotel name and room pricing across every screenshot.
